---

## 1. НАЗНАЧЕНИЕ СИСТЕМЫ

Система предназначена для **проактивного управления** добывающими скважинами с электроприводными насосами (ЭЦН) через **автоматический анализ телеметрии и формирование управляющих сигналов**.

**Ключевые возможности:**
- Автоматический сбор данных телеметрии в реальном времени
- Предиктивная аналитика с использованием ML
- Непосредственное формирование управляющих сигналов на оборудование
- Веб-интерфейс для мониторинга и ручного управления

## 3. ТРЕБОВАНИЯ К ДАННЫМ

### 3.1. Телеметрия в реальном времени (получается через FGI/REST)
1. **Электрические параметры** (каждые 5-30 секунд):
   - Максимальный ток по фазам
   - Среднее фазное напряжение
   - Полная/активная мощность на двигателе
   - Токи фаз A, B, C
   - Активные токи фаз
   - Сопротивление изоляции
   - Cos φ двигателя
   - Напряжения фаз и линейные
   - Дисбаланс токов и напряжений
   - Частота питания ПЭД
   - Потребление электроэнергии (сутки)

2. **Механические параметры**:
   - Температура жидкости на приеме насоса
   - Давление на приеме насоса
   - Температура двигателя
   - Вибрация по осям X, Y
   - Загрузка (%)

### 3.2. Технические характеристики оборудования (статичные данные)
1. **Электродвигатель**:
   - Тип, модель, производитель
   - Номинальная мощность (кВт)
   - Номинальное напряжение (В)
   - Номинальный ток (А)
   - КПД, cos φ номинальный
   - Класс изоляции
   - Степень защиты (IP)
   - Масса, габариты

2. **Насос ЭЦН**:
   - Тип, модель, производитель
   - Номинальная подача (м³/сут)
   - Номинальный напор (м)
   - КПД насоса
   - Количество ступеней
   - Диаметр корпуса
   - Материал проточной части

3. **Скважина**:
   - Глубина скважины
   - Диаметр колонны
   - Глубина спуска насоса
   - Динамический уровень
   - Статический уровень
   - Дебит пластовый

## 7. API СПЕЦИФИКАЦИЯ

### 7.1. Основные эндпоинты:
```
GET    /api/wells/                    # Список скважин
GET    /api/wells/{id}/               # Данные скважины
GET    /api/wells/{id}/telemetry/     # Текущая телеметрия
GET    /api/wells/{id}/history/       # Исторические данные
POST   /api/wells/{id}/control/       # Управляющая команда
GET    /api/wells/{id}/predictions/   # Прогнозы ML
GET    /api/analytics/anomalies/      # Обнаруженные аномалии
POST   /api/control/batch/            # Групповое управление
```

### 7.2. Формат управляющей команды:
```json
{
  "well_id": "WELL-001",
```
